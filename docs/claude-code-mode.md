# 클로드 코드 모드 (Claude Code Mode)

> **개발 전용(운영 미노출)** — 클로드 코드 모드는 ADR-0009 의 개발 티어 전용 기능입니다. 운영(`APP_ENV=prod` / `FEATURE_CLAUDE_CODE=off`)에서는 `claude_code`·`workspace` 라우터가 등록되지 않고, chat 라우터의 직행 분기에서도 `claude_code_mode=true` 면 403("이 환경에서는 Claude Code 모드를 사용할 수 없습니다.")으로 거부되며 `claude_code` 도구가 항상-켜짐 세트에서 제외됩니다(`router.py` `_ALWAYS_ON_TOOLS` discard). 프론트의 슬래시 메뉴/배너에도 노출되지 않습니다.

채팅 인터페이스 안에서 **실제 파일을 읽고/쓰고/실행**할 수 있는 코딩 전용 모드입니다. 일반 채팅 모드가 "대화 위주"라면, 클로드 코드 모드는 "작업 위주"로 동작하며 가상 작업공간(workspace)을 가집니다.

---

## 클로드 코드 모드란

클로드 코드 모드는 Anthropic의 [Claude Code](https://docs.claude.com/en/docs/claude-code/overview) 엔진을 챗봇 내부에 임베드한 실행 모드입니다. 일반 채팅과 가장 큰 차이는 다음 세 가지입니다.

- **파일 시스템**: 대화마다 격리된 작업공간(sandbox)이 생성되어, 코드/문서/설정 파일을 그 안에서 직접 편집할 수 있습니다.
- **도구 호출**: `Read`, `Write`, `Edit`, `Bash`, `Grep` 등 표준 Claude Code 툴셋을 그대로 사용합니다.
- **에이전트 루프**: 모델이 스스로 여러 단계를 계획·실행·검증하며, 사용자는 중간 결과를 스트리밍으로 받아볼 수 있습니다.

따라서 "리팩터링", "테스트 작성", "버그 재현", "마이그레이션 스크립트 생성" 같은 다단계 작업에 적합합니다.

## 켜고 끄는 법

### 1) 슬래시 메뉴로 켜기

채팅 입력창에서 `/` 를 누르면 명령 메뉴가 열립니다. **"클로드 코드 모드 (고급)"** 항목(id: `claude-code`)을 선택하면 즉시 모드가 켜지고, 상단에 **accent 보더의 모드 배너**가 표시됩니다. `/claude` 처럼 이어서 입력해 항목을 필터링할 수도 있습니다.

모드는 대화마다 **초기 OFF** 입니다 — 진입할 때마다 명시적으로 켜야 하며, 이전 작업이 있던 대화는 아래 "돌아가기 CTA" 로 모드까지 한 번에 복귀할 수 있습니다.

### 2) 배너의 토글로 끄기

상단 배너 우측의 **"모드 끄기"** 버튼을 누르면 일반 채팅 모드로 복귀합니다. 슬래시 메뉴의 같은 항목("클로드 코드 모드 끄기")을 선택해도 토글됩니다.

> 모드를 꺼도 작업공간은 사라지지 않습니다. 다시 켜면 마지막 상태에서 이어집니다.

## 처음 켜면 — 3단계 튜토리얼

모드를 처음 켜면(사용자별 1회) 화면 우하단에 **3단계 워크스루**가 자동으로 열립니다(`frontend/features/chat/ClaudeCodeTutorial.tsx`). 모드가 켜지는 *모든 경로*(슬래시 토글·복귀 CTA)에서 미완료 사용자에게만 노출됩니다.

| 단계 | 내용 |
| --- | --- |
| ① 🚀 직행 모드 | 모든 입력이 Claude Code 로 곧장 전달 — 일반 채팅의 의도 분류·짧은 입력 차단은 꺼지고, 내 Anthropic API 키(BYOK)로 실행됨 |
| ② 📂 오른쪽 작업공간 패널 | 파일 트리 → ▶ 실행 → 👁 미리보기 흐름 안내. 이 단계 동안 실제 작업공간 패널이 accent 로 하이라이트됨 |
| ③ ⌨️ 명령·옵션·첨부 | `/` 슬래시 카탈로그(`/review`, `/plan`, `/loop` …), 배너의 모델·추론 강도 셀렉터, 📎 파일·폴더 첨부(구조 보존) |

- 완주하거나 건너뛰면 사용자별 `localStorage`(`chat:ccTutorial:done:<user>`)에 기록되어 다시 자동으로 뜨지 않습니다.
- 다시 보려면: 모드 배너 타이틀 옆 **`?` 버튼** 또는 `/` 메뉴의 **튜토리얼(`/powerup`)** 항목을 사용합니다.
- 모드에서 첫 메시지를 보내면 온보딩 퀘스트 **"코드 작업 맡겨보기"**(`tried_claude_code`)가 완료 처리됩니다.

## 추론 강도 (Reasoning Effort)

모드 배너의 셀렉트(또는 `/` 메뉴의 **추론 강도**(`/effort`) 항목)로 추론 강도를 선택할 수 있습니다. 이는 실제 claude CLI 의 `--effort` 플래그로 매핑되며, 유효값은 **5단계**(`low | medium | high | xhigh | max`)입니다. 단계가 높을수록 모델이 더 많은 추론 토큰을 쓰며, 응답 시간과 비용이 늘어납니다.

| 단계 | 권장 용도 |
| --- | --- |
| `low` | 단순 문답·포맷 변환·짧은 코드 스니펫(가장 빠름, 토큰 최소) |
| `medium` | 다중 파일 변경, API 설계 — **기본값** |
| `high` | 시스템 설계, 마이그레이션 계획(깊은 추론) |
| `xhigh` | 난도 높은 리팩터링·디버깅(매우 깊은 추론) |
| `max` | 대규모 아키텍처 변경, 보안 감사(최대 thinking budget) |

기본값은 `medium` 입니다. 명시적으로 바꾸지 않는 한 대화 내내 유지됩니다.

> 과거 UI 에 있던 `none`/`minimum` 단계는 실제 CLI 에 존재하지 않는 가짜 레벨이라 제거되었습니다(가짜 값 입력 시 `medium` 으로 보정). 옵션 매핑 상세는 [Claude Code CLI 옵션 매핑](./claude-code-cli.md) 참고.

## Anthropic 키 등록 (마이페이지 BYOK)

클로드 코드 모드는 **BYOK(Bring Your Own Key)** 방식으로도 사용 가능합니다.

1. 우측 상단 아바타 → **마이페이지**
2. **"API 키"** 섹션(`ApiKeysSection`)
3. **Anthropic API Key** 항목에 `sk-ant-...` 로 시작하는 키를 입력 후 저장
4. 저장된 키는 서버에서 **Fernet**(AES-128-CBC + HMAC-SHA256, ADR-0005)으로 애플리케이션 레벨 암호화되어 보관됩니다 — 평문은 로그·응답 어디에도 노출되지 않고, 조회 시 마스킹된 fingerprint 만 반환됩니다

BYOK 가 설정되어 있으면 클로드 코드 모드의 호출이 **개인 키 quota** 로 청구되며, 회사 공용 키 한도와 분리됩니다. 키를 삭제하면 자동으로 공용 키로 폴백합니다.

## 파일·폴더 첨부 — 디렉토리 구조 보존

클로드 코드 모드에서는 단일 파일뿐 아니라 **폴더째 첨부**할 수 있고, 폴더의 상대 경로 구조가 작업공간에 그대로 재현됩니다(2026-06-10 — 이전에는 모든 첨부가 basename 으로 평탄화되어 `src/app/page.tsx` 가 `page.tsx` 로 떨어졌습니다).

- **📎 폴더 첨부 버튼** — `input[webkitdirectory]` 로 폴더를 선택하면 `File.webkitRelativePath` 기준으로 구조를 보존해 첨부합니다.
- **드래그&드롭** — 폴더를 떨어뜨리면 재귀 탐색합니다. `node_modules`, `.git`, `.next`, `dist`, `build`, `__pycache__`, `.venv` 디렉토리는 자동 제외되고, 한 번에 **최대 300개** 파일까지만 첨부됩니다(초과분 제외 안내).
- **프리뷰** — 첨부 목록에 상대 경로(`src/app/page.tsx`)가 그대로 표시됩니다.
- **전송 스키마** — 각 첨부는 `{filename, b64, relpath?}` 로 전달되고(`FileAttachment.relpath`), 백엔드는 `app/core/pathsafe.safe_relative_path` 게이트를 통과한 경로에만 구조 보존을 적용합니다. 절대경로·`..` 토큰 등 traversal 위험 경로는 **basename 폴백**으로 격하되며, 최종 경로는 `resolve()` + `commonpath` 로 작업공간 격리를 한 번 더 검증합니다. 상세: [Claude Code CLI 옵션 매핑](./claude-code-cli.md)의 보안 모델.

## 작업공간 영속 — 같은 대화 다시 열면 파일 자동 복원

각 대화에는 고유한 작업공간 ID 가 부여되고, 모드 중 생성·수정된 파일은 서버의 대화별 영속 디렉토리(`data/workspaces/{workspace_id}/files/`, `workspace_manager.py`)에 보존됩니다.

- 브라우저를 닫았다가 다시 열어도 **파일 트리·내용**이 그대로 복원됩니다.
- 다른 기기에서 같은 계정으로 접속해도 동일한 작업공간을 보게 됩니다(서버 측 저장).

> 작업공간은 대화별로 **완전히 격리**됩니다. A 대화에서 만든 파일은 B 대화에서 보이지 않습니다.

## 채팅 모드에서 결과로 돌아가기

클로드 코드 모드에서 어떤 작업을 한 뒤 모드를 끄면, 채팅창 하단에 다음과 같은 CTA(Call-To-Action) 카드가 표시됩니다.

> 🤖 **이전 클로드 코드 작업으로 돌아가기** · 파일 4개 · 도구 6회 실행 ↗

이 카드를 누르면 직전 작업공간이 그대로 다시 열리고, 모드도 자동으로 켜집니다. 채팅 모드와 코딩 모드를 자유롭게 오가며 같은 맥락을 유지할 수 있도록 설계된 장치입니다.

CTA 카드는 **이 대화에서 클로드 코드가 실제로 돌았다는 신호**가 있을 때만 노출됩니다(대화 스코프 `sessionStorage`, 탭을 닫으면 폐기).

- 사용자가 이 대화에서 슬래시로 클로드 코드 모드를 켠 적이 있거나,
- 이 대화의 스트림에서 `agent_done` 이벤트가 한 번이라도 왔을 때

> `workspace_id` 나 파일 개수는 신호로 쓰지 않습니다 — 백엔드가 모든 도구 사용 대화에 기본 워크스페이스를 lazy-bind 하므로 일반 대화에서도 true 가 되어 CTA 가 새던 회귀(2026-06-04)가 있었습니다.
