# 클로드 코드 모드 (Claude Code Mode)

채팅 인터페이스 안에서 **실제 파일을 읽고/쓰고/실행**할 수 있는 코딩 전용 모드입니다. 일반 채팅 모드가 "대화 위주"라면, 클로드 코드 모드는 "작업 위주"로 동작하며 가상 작업공간(workspace)을 가집니다.

---

## 클로드 코드 모드란

클로드 코드 모드는 Anthropic의 [Claude Code](https://docs.claude.com/en/docs/claude-code/overview) 엔진을 챗봇 내부에 임베드한 실행 모드입니다. 일반 채팅과 가장 큰 차이는 다음 세 가지입니다.

- **파일 시스템**: 대화마다 격리된 작업공간(sandbox)이 생성되어, 코드/문서/설정 파일을 그 안에서 직접 편집할 수 있습니다.
- **도구 호출**: `Read`, `Write`, `Edit`, `Bash`, `Grep` 등 표준 Claude Code 툴셋을 그대로 사용합니다.
- **에이전트 루프**: 모델이 스스로 여러 단계를 계획·실행·검증하며, 사용자는 중간 결과를 스트리밍으로 받아볼 수 있습니다.

따라서 "리팩터링", "테스트 작성", "버그 재현", "마이그레이션 스크립트 생성" 같은 다단계 작업에 적합합니다.

---

## 켜고 끄는 법

### 1) 슬래시 명령으로 켜기

채팅 입력창에서 다음을 입력합니다.

```
/클로드 코드 모드
```

또는 영문 alias 인 `/claude-code` 도 동일하게 동작합니다. 입력 즉시 상단에 **보라색 배너**가 표시되며, 현재 대화의 모드가 클로드 코드로 전환됩니다.

### 2) 배너의 토글로 끄기

상단 배너 우측의 **"모드 끄기"** 버튼을 누르면 일반 채팅 모드로 복귀합니다. 같은 슬래시 명령을 한 번 더 입력해도 토글됩니다.

> 모드를 꺼도 작업공간은 사라지지 않습니다. 다시 켜면 마지막 상태에서 이어집니다.

---

## 6단계 사고 강도 (Thinking Effort)

모드 헤더의 슬라이더 또는 `/사고강도` 명령으로 6단계 중 하나를 선택할 수 있습니다. 단계가 높을수록 모델이 더 많은 추론 토큰을 쓰며, 응답 시간과 비용이 늘어납니다.

| 단계 | 권장 용도 |
| --- | --- |
| `none` | 단순 문답·포맷 변환·짧은 코드 스니펫 |
| `minimum` | 1파일 수정, 명확한 버그 픽스 |
| `low` | 소규모 리팩터링, 테스트 보강 |
| `medium` | 다중 파일 변경, API 설계 |
| `high` | 시스템 설계, 마이그레이션 계획 |
| `max` | 대규모 아키텍처 변경, 보안 감사 |

기본값은 `medium` 입니다. 명시적으로 바꾸지 않는 한 대화 내내 유지됩니다.

## Anthropic 키 등록 (마이페이지 BYOK)

클로드 코드 모드는 **BYOK(Bring Your Own Key)** 방식으로도 사용 가능합니다.

1. 우측 상단 아바타 → **마이페이지**
2. 좌측 메뉴 **"API 키"** 탭
3. **Anthropic API Key** 항목에 `sk-ant-...` 로 시작하는 키를 입력 후 저장
4. 저장된 키는 서버에서 AES-GCM 으로 암호화되어 보관됩니다(평문 노출 X)

BYOK 가 설정되어 있으면 클로드 코드 모드의 호출이 **개인 키 quota** 로 청구되며, 회사 공용 키 한도와 분리됩니다. 키를 삭제하면 자동으로 공용 키로 폴백합니다.

## 작업공간 영속 — 같은 대화 다시 열면 파일 자동 복원

각 대화에는 고유한 작업공간 ID 가 부여되고, 모드 중 생성·수정된 모든 파일은 객체 스토리지에 스냅샷됩니다.

- 브라우저를 닫았다가 다시 열어도 **파일 트리·내용·실행 이력**이 그대로 복원됩니다.
- 다른 기기에서 같은 계정으로 접속해도 동일한 작업공간이 마운트됩니다.
- 좌측 사이드바의 대화 항목 옆 **"폴더 아이콘"** 으로 작업공간 보존 여부를 확인할 수 있습니다.
- 30일간 활동이 없으면 작업공간은 콜드 스토리지로 이동하며, 다시 열 때 수 초가 소요될 수 있습니다.

> 작업공간은 대화별로 **완전히 격리**됩니다. A 대화에서 만든 파일은 B 대화에서 보이지 않습니다.

## 채팅 모드에서 결과로 돌아가기

클로드 코드 모드에서 어떤 작업을 한 뒤 모드를 끄면, 채팅창 하단에 다음과 같은 CTA(Call-To-Action) 카드가 표시됩니다.

> **이전 클로드 코드 작업으로 돌아가기**
> _마지막 변경: 12분 전 · 파일 4개_

이 카드를 누르면 직전 작업공간이 그대로 다시 열리고, 모드도 자동으로 켜집니다. 채팅 모드와 코딩 모드를 자유롭게 오가며 같은 맥락을 유지할 수 있도록 설계된 장치입니다.

CTA 카드는 다음 조건에서만 노출됩니다.

- 동일 대화 내에서 최근 1시간 이내 코드 모드 활동이 있을 때
- 작업공간에 1개 이상의 변경 파일이 존재할 때

## 키보드 단축키 표

| 동작 | macOS | Windows / Linux |
| --- | --- | --- |
| 클로드 코드 모드 토글 | `⌘` + `Shift` + `K` | `Ctrl` + `Shift` + `K` |
| 사고 강도 +1 단계 | `⌘` + `]` | `Ctrl` + `]` |
| 사고 강도 −1 단계 | `⌘` + `[` | `Ctrl` + `[` |
| 작업공간 파일 트리 열기 | `⌘` + `B` | `Ctrl` + `B` |
| 현재 작업 중단 (Stop) | `Esc` | `Esc` |
| 마지막 명령 재실행 | `⌘` + `R` | `Ctrl` + `R` |
| 이전 작업공간으로 점프 | `⌘` + `Shift` + `←` | `Ctrl` + `Shift` + `←` |
| 슬래시 명령 팔레트 | `/` | `/` |
| 멀티라인 입력 토글 | `⌘` + `Enter` | `Ctrl` + `Enter` |
| 전송 | `Enter` | `Enter` |

> 단축키는 **마이페이지 → 환경설정 → 키보드** 에서 자유롭게 재정의할 수 있으며, JSON 으로 import/export 도 가능합니다.
