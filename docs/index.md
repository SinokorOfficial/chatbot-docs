# 장금상선 그룹 챗봇 · 기술 가이드

장금상선 / 흥아라인 **AI 파트** 가 운영하는 사내 그룹 챗봇 시스템의 **사용 매뉴얼**·**기술 문서**·**기여 가이드** 를 한 곳에 모은 공개 사이트입니다. 코드 자체는 비공개 리포에서 관리하며, 이 사이트는 문서만 동기화합니다.

---

## 한눈에 보는 시스템

사내 팀이 자기들 도구·문서·스킬·워크플로를 모아 **그룹 단위로 공유할 수 있는 AI 챗봇 플랫폼**.

| 영역 | 무엇을 |
|---|---|
| **챗봇** | 팀 공용/개인용 챗봇 프로필 — 모델·시스템 프롬프트·문서·도구 묶음 |
| **문서 RAG** | PDF / TXT 업로드 → 자동 청크·임베딩 → 챗봇이 인용 가능 |
| **도구 마켓** | 웹 검색·코드 인터프리터·외부 MCP 도구를 슬러그 단위로 등록·재사용 |
| **스킬 마켓** | 자주 쓰는 프롬프트·체크리스트·플로우를 카드로 공유, 좋아요/포크 |
| **워크플로** | 트리거 → 도구 → LLM → HTTP 노드를 2D 캔버스·3D 캔버스에서 시각 편집 |
| **Q&A** | 챗봇별 FAQ 게시판 — 사용자가 질문, AI/관리자가 답변 |
| **공지·매뉴얼·사용량** | DB 기반 공지, 마이페이지의 USD/KRW 사용량 요약 |
| **권한·감사** | RBAC 3단계(member / team_auditor / team_admin / super_admin) + 감사 로그 |

---

## 사용자라면 — 사용 매뉴얼

화면 위주로 정리한 [사용 매뉴얼](user_manual.md) 부터 읽으세요. 로그인·챗봇 만들기·문서 업로드·Q&A 작성까지 단계별로 안내합니다.

## 시스템 구조가 궁금하다면

먼저 [아키텍처 개요](architecture.md) → [모듈 맵](modules.md) → [ERD](erd.md) 순서로 보면 큰 그림이 빠르게 잡힙니다. 결정 배경은 [ADR](architecture-decisions.md) 에 모았습니다.

## 기능을 추가·수정한다면 — 문서화 규칙

- **모든 기능 추가/변경은 `docs/` 안에 동시에 반영합니다** (코드만 머지하면 안 됨).
- 새 기능이면 해당 영역 문서를 갱신하거나 새 `.md` 를 추가하고, `mkdocs.yml` 의 `nav:` 에 등록합니다.
- 변경 이력은 [changelog.md](changelog.md) 에 날짜 + Added/Changed/Removed 섹션으로 추가합니다.
- DB 스키마가 바뀌면 [erd.md](erd.md) 의 Mermaid ERD 와 컬럼 표를 함께 갱신합니다.
- 본 사이트는 `docs/` 또는 `mkdocs.yml` 이 main 에 푸시되면 GitHub Actions 가 자동 빌드/배포합니다.
